# blog-samples

Generic samples I might refer to on my blog or simply share with people where relevant.  Low bar.  This is not meant for re-use but just to show some things can be one or few shotted easily.  Just go try them out.  Throwaways of course, are not production code.  This is not about production code.

# AI

Many samples will be fast, practical tests or prototypes built in short sessions.  Meant to showcase non-coders that they should try stuff with AI to get familiar.  You don't get the feel about what's possible only by using what others build.  Build things yourself.

Now, if you need to collaborate or share, other principles may apply but getting the feel is important.

Many will feature local LLMs because that is the ultimate measure of control but same principles apply for connecting to LLMs on the cloud.

## Characteristics

- Time-boxed implementation (5, 10, 30, 60 minutes max)
- In the past, we would've almost shared gists or pastebins.
- Minimal dependencies and setup
- Iterative, conversational workflow

## Expectations

Prototypes, not production systems.  Don't put any of these in production.

## Try One In 5 Minutes

1. Pick any sample directory.
2. Follow its local `README.md`.
3. Run the first command.
4. Modify one thing and rerun.

## Why This Repo Exists

Prototyping was never very hard but fighting configuration and knowing how to break down problems could be a learned skill.  Not it's not just easy but an utterance can become a live test in a few minutes, and the back and forth might allow you to tailor it just right to be useful to you or prove some idea for an actual project (which you probably want to start clean with some written specifications and desigining with some more intent).

Where it makes sense, I'll just just share a link.  

None of these things are meant to be brilliant or unique.  It's just quite sharing of code with friends, co-workers or whoever might benefit so the random-tests don't pollute proper directed projects.
